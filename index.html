commande git

git init

On inisialise le dépot (repository) gràce à la commande git init

git config

permet de déinir la configuration de git sur votre mashine.cette commande n'est à utiliser que lorque souhaitez la modifier et non à chaque projet

On peux par exemple la première fois faire git config --global (on modifier la configuration)

Git add

Permet d'ajoute la nouveaux version des fichiers

La commande utilisée est git add (nom_du_fichier)

Une fois utilisée,les fichiers deviennent added

git add . est la commande générique qui permet d'ajouter tous les fichiers

git commit

Une fois les fichiers ajoutés on utilise git commit pour valider la version à laquelle on ajoute une description

La commande est git commit -m "Description"

git log

qui permet d'optinir les log des commit donc l'historique de notre projet

Chaque commit a différentes information les infos de l'auteur,un id unique et pour le commit le plus récent => HEAD->master

git branch

qui permet d'afficher et de gérer les branch

git branch affiche les branch

git branch -M modifier le nom de d'une branch

git branch -C ajoute une branch

git branch -D supprime une branch

git chekout/git switch

Permet de naviguer entre nos branch

La commande est git checkout (nom_de_la_branch)

git merge

Une fois qu'un travail est finir sur une branch, on veut associer le résultat à une autre branch (ex:master)

On se prend sur la branch qui va recevoir les infos puis on utiliser git merge (nom de la branch)

La branch qui recoit s'actualise et récupère les nouvelles infos
